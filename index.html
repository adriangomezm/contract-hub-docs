Contract Hub User Documentation

The complete guide to managing digital contracts and e-signatures for high-volume recruitment.

Introduction

Contract Hub is Talkpush's native e-signature platform purpose-built for high-volume hiring. It replaces third-party signing tools like DocuSign by providing an integrated, mobile-first contract experience delivered directly through channels candidates already use — WhatsApp, SMS, and web links.

Contract Hub enables HR teams to create contract templates, automatically match them to the right candidates using envelopes with smart matching logic, and track signing progress in real time — all from a single multi-tenant admin interface.

What Contract Hub Does

  • Template Management — Upload PDF or DOCX documents, visually place fillable fields, and configure reusable contract templates.
  • Envelope Grouping — Bundle multiple templates together and assign them to campaigns with AND/OR matching logic.
  • Smart Matching — Automatically assign the right contract to candidates based on campaign and attribute filters with eight flexible operators.
  • Mobile-First Signing — Candidates receive a link and sign on any device, no app or account required.
  • AI Assistance — An in-document chatbot helps candidates understand their contract in plain language.
  • Real-Time Tracking — Monitor signing progress, send reminders, and export audit trails.
  • Multi-Tenant Architecture — Manage multiple companies from a single account with role-based access control.
  • Legal Compliance — ESIGN Act, UETA, and eIDAS compliant with SHA-256 tamper-evident audit trails.

Creating a Contract

Click the New Contract button on the Contracts page to start the creation wizard.

  1. Upload Document
    Drag and drop a .pdf or .docx file into the upload zone, or click to browse your files. The maximum file size is 20 MB. The document will be converted and rendered as a preview in the next steps.
  2. Contract Details
    Enter the contract name, select a category (e.g., Offer Letter, Employment Contract, NDA), and add an optional description. This metadata helps you organize and search for templates later.
  3. Signers
    Define who needs to sign the document. You can configure multiple signers (e.g., candidate + HR manager) and map each signer's name and email to Talkpush candidate attributes.
  4. Edit & Add Fields
    This opens the full-screen Field Overlay Editor where you visually place interactive fields directly on the document pages. See the Field Overlay Editor section for details.

Template Builder

The Template Builder is where you edit the metadata and configuration of a contract template. It covers the contract details, signer definitions, and overall template settings. This is distinct from the Field Overlay Editor, which is specifically for placing fields on the document visually.

Template Metadata

At any time, you can update:

  • Contract name — The internal name visible to administrators.
  • Category — The document type classification.
  • Description — Optional notes about the template's purpose.
  • Version — Automatically incremented when changes are saved.

Each template can have one or more signers. For each signer, you can specify:

  • Signer label — A display name for the signer role (e.g., "Candidate", "HR Manager").
  • Name attribute key — The Talkpush candidate attribute that provides the signer's name.
  • Email attribute key — The Talkpush candidate attribute that provides the signer's email address.

Fields can have validation constraints to ensure data quality:

  • Max Length — Maximum character count for text fields.
  • Pattern — Regular expression for format enforcement (e.g., email format).
  • Min/Max Value — Numeric range limits for number fields.
  • Date Format — Display format for date fields (e.g., MM/DD/YYYY).

Writeback

When a writebackAttributeKey is configured on a field, the value entered by the candidate during signing is written back to their Talkpush profile. This keeps candidate data synchronized between Contract Hub and the Talkpush CRM.

Envelopes

Envelopes are the primary way to organize and assign contracts to candidates. An envelope is a group of one or more templates combined with campaign assignments and attribute-based matching rules.

Why Use Envelopes?

Instead of assigning contracts individually, envelopes let you bundle related documents together and define a single set of rules for when they should be sent. For example, an envelope called "US Full-Time Onboarding" might contain an offer letter, an NDA, and a tax compliance form — all assigned to candidates who match specific campaign and attribute criteria.

Envelope Structure

Each envelope contains:

  • Name & Description — A human-readable label and optional description for internal reference.
  • Status — Active, Draft, or Archived (same as template statuses).
  • Templates — One or more contract templates included in this envelope.
  • Campaigns — Which Talkpush campaigns trigger this envelope.
  • Attribute Filters — Conditions that candidates must match to receive this envelope.
  • Match Logic — Whether all attribute filters must match (AND) or just one (OR).

Creating an Envelope

  1. Click "New Envelope"
    From the Envelopes page, click the New Envelope button to start configuration.
  2. Set Details
    Enter a name and optional description for the envelope.
  3. Add Templates
    Click Add Template to open the template selection dialog. Choose one or more contract templates to include.
  4. Assign Campaigns
    Click Add Campaign to select which Talkpush campaigns should trigger this envelope.
  5. Configure Attribute Filters
    Add conditions using the 14 available attributes and 8 filter operators. Choose AND or OR logic to control how multiple filters are evaluated.

Match Logic: AND vs. OR

  • AND (All must match) — The candidate must satisfy every filter to receive the envelope. Use this for precise targeting. Example: Country is "Philippines" AND Employment Type is "Full-Time".
  • OR (Any can match) — The candidate needs to match at least one filter. Use this for broader distribution. Example: Department is "Engineering" OR Department is "Product".

Envelope Dashboard

The Envelopes page displays all envelopes in a searchable grid with status filters. Each envelope card shows the number of templates, linked campaigns, attribute filters (with match logic badge), and session statistics (total sessions created vs. completed).

Matching Rules

Matching Rules determine which candidates automatically receive a specific contract template. Rules are evaluated when a candidate enters a campaign, and the system checks whether they match the configured criteria.

Match Logic

For each template or envelope, you can choose how multiple rules are evaluated:

  • Match ALL (AND) — Every rule must be satisfied. Strictest matching.
  • Match ANY (OR) — At least one rule must be satisfied. Broadest matching.

Campaign Filters

Assign specific Talkpush campaigns to a contract template. Only candidates in the selected campaigns will be considered for this contract.

Attribute Filters

Beyond campaign matching, you can add attribute-based conditions using the following operators:

OperatorDescriptionExample

Tip: Use is_empty and is_present operators to create fallback contracts. For example, send a generic offer letter when the "Department" attribute is_empty, and a department-specific letter when it is_present.

Use the toolbar to narrow down your view:

  • Status filter — Filter by Completed, In Progress, Pending, Viewed, Declined, Expired, or Voided.
  • Search — Find sessions by candidate name or email.
  • Date presets — Quick filters for Today, Last 7 Days, Last 30 Days, or a custom date range.

Candidate Signing Experience

Candidates interact with Contract Hub through a dedicated mobile-first signing interface. No account or app is required — candidates simply click a link sent via WhatsApp, SMS, or email and complete the signing process in their browser.

The candidate experience is designed to be simple, accessible, and guided:

  1. Welcome Screen — Introduction and instructions.
  2. Read the Contract — Scroll through the full document.
  3. Ask the AI Assistant — Optionally chat with an AI that explains the contract in plain language.
  4. Fill Fields & Sign — Complete required fields and add their signature.
  5. Confirmation — Receive a completion screen and download option.

Welcome Screen

When a candidate opens their signing link, they are greeted with a welcome screen that shows the company logo, a brief explanation of what they'll be doing, and a Get Started button.

Reading the Contract

The contract is displayed in a scrollable, mobile-optimized viewer. Candidates can read through the entire document before filling any fields. The interface highlights which sections require attention.

AI Contract Assistant

An AI-powered chatbot is available throughout the signing experience. Candidates can tap the chat icon to ask questions about any part of the contract. The AI explains clauses, defines legal terms, and clarifies obligations — all in plain, friendly language.

Tip: The AI Assistant only explains the contract — it cannot modify the document or provide legal advice.

Signing the Contract

After reading the contract, the candidate proceeds to the signing step. They fill in any required fields (text, dates, checkboxes) and draw or type their signature in the designated signature area. All fields pre-filled from Talkpush data are visible and can be reviewed.

Once all required fields are complete, the candidate taps Complete Signing to submit the document. The system records a timestamped audit event for the signature.

Completion

After signing, the candidate sees a confirmation screen with a success message. They can download a copy of the signed document for their records. The session status is updated to Completed and the audit trail is finalized.

Security & Compliance

Contract Hub is built with security and legal compliance at its core, leveraging Supabase for backend infrastructure with enterprise-grade security features.

Authentication & Access

  • Google SSO — Secure single sign-on via Google Workspace with OAuth 2.0.
  • Email/Password — Supabase-powered authentication with secure password hashing.
  • Role-Based Access Control — Three-tier RBAC with route-level protection. Protected routes enforce role checks before page access.
  • Multi-Tenant Isolation — Each company workspace is fully isolated. Users can only access data within their assigned companies.
  • Invitation-Only Registration — No self-registration. Users must be invited by an administrator.

Signing Security

  • Unique Signing Links — Each signing session generates a unique, time-limited token-based URL.
  • SHA-256 Audit Trail — Every action is logged with cryptographic hashing to ensure tamper evidence.
  • IP & Device Logging — The audit trail captures IP addresses and user-agent strings for each event.
  • Session Expiration — Signing links automatically expire after the configured deadline.
  • Voiding — Administrators can void active sessions, immediately invalidating the signing link.

Legal Compliance

  • ESIGN Act (United States) — Electronic signatures are legally equivalent to handwritten signatures.
  • UETA (Uniform Electronic Transactions Act) — Adopted in 49 U.S. states.
  • eIDAS (European Union) — Compliant with EU electronic identification and trust services regulation.
  • Data Encryption — All data is encrypted in transit (TLS 1.2+) and at rest.

Frequently Asked Questions

What file formats can I upload?

Contract Hub supports PDF and DOCX (Microsoft Word) documents. The maximum file size is 20 MB. Both formats are rendered in the Field Overlay Editor for visual field placement.

Can I send multiple contracts at once?

Yes. Use Envelopes to group multiple templates together. When a candidate matches the envelope's rules, all templates in the envelope are sent as part of a single signing session. Candidates see their progress as "2/3 documents" etc.

What's the difference between the Template Builder and the Field Overlay Editor?

The Template Builder is for editing contract metadata (name, category, description) and configuring signers. The Field Overlay Editor is the visual drag-and-drop tool where you place interactive fields directly on the document pages. They are accessed from different buttons on the Contracts page.

What happens when a session expires?

The signing link becomes inactive and the candidate can no longer access the document. The session status changes to Expired. To give the candidate more time, create a new session with a fresh link.

Can candidates sign on mobile?

Yes. The signing experience is fully mobile-optimized. Candidates can read, fill fields, and sign contracts on any smartphone or tablet browser without installing an app.

Are electronic signatures legally valid?

Yes. Contract Hub complies with the ESIGN Act, UETA, and eIDAS regulations, making electronic signatures legally equivalent to handwritten signatures in the United States, Europe, and most international jurisdictions.

How do I switch between companies?

Click on the company name in the left sidebar (shown with a building icon). This takes you back to the Company Selector where you can choose a different company workspace. All data is scoped to the selected company.

What's the difference between AND and OR matching in envelopes?

AND means the candidate must satisfy every filter condition. OR means the candidate only needs to match at least one filter. AND is stricter (for precise targeting), OR is broader (for wider distribution).

How does data autofill work?

Fields configured with the Talkpush Attribute data source automatically pull values from the candidate's Talkpush profile (e.g., name, email, address). You can optionally set isEditableWhenAutofilled to let the candidate modify the pre-filled value, and fallbackValue to provide a default when the attribute is empty.

Can I use Google SSO to sign in?

Yes. Click Continue with Google on the login page to authenticate using your Google Workspace account. Your Google account must be authorized by your administrator. If you see an "not authorized" error, contact your admin for an invitation.

What is a Company User and what can they access?

A Company User is a limited-access role intended for recruiters or coordinators. They can view and track Signing Sessions and access the Help Center, but cannot manage templates, envelopes, users, or settings. This keeps the interface simple for team members who only need to monitor contract progress.

Can I write field values back to Talkpush?

Yes. Configure the writebackAttributeKey on any field to automatically sync the candidate's input back to their Talkpush profile after signing. This is useful for capturing new information like emergency contacts, bank details, or tax IDs that are entered during the contract signing process.
